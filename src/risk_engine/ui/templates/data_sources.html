This page describes where the tool looks for records and what backs each flag. It is purely descriptive. Nothing here scores or ranks a case.

Record sources (the court-record universe)

When you flag an intake, the engine matches the applicant to public court records by name and conviction year from the source you pick on the intake form. Today the records are appellate opinions; trial dockets and transcripts are not wired up yet, so they show as honest gaps rather than "clean".

"On demand" sources fetch opinion text live from the CourtListener REST API when you flag a case (network + API token required). Offline fixtures are bundled with the app and need no network.

Bulk offline snapshot (for mass matching)

For large-scale linking without hitting the API, a quarterly CourtListener bulk snapshot is staged on the server (~53 GB): the full opinions corpus plus the opinion-cluster metadata (captions, courts, dates). The offline matcher streams it once and keeps text only for records that actually match — a bulk match means exactly the same thing as an API match (same name + year scoring).

Discredited-method references

The exoneration corpus is the National Registry of Exonerations set, used to calibrate which element categories are worth flagging. It is a reference/label set, kept strictly separate from live intakes and never commingled with them. The discredited-method references are the case-independent scientific citations (NAS 2009, PCAST 2016, FBI/DOJ reviews) that every discredited forensic method flag is checked against — the reason a flag is verifiable regardless of guilt or innocence.

Demo records (source demo_marvel) are de-identified and demonstration-only; they are excluded from the reference corpora and never used for training or calibration.

Debunked / discredited forensic science (the method reference)

This is the data source behind every discredited forensic method flag. When a method named here appears in a case record, the flag fires and carries the citation below so an attorney can verify it. The tier is a per-method label grounded in a named authority — never a score: A formally invalidated or abandoned, B unvalidated per NAS 2009 / PCAST 2016 but still in use, C contested or evolving.
